Для объекта User с полями String name и int age размер составит 72 байта(заголовок+ объекта 16, примитив int 4
и String с его внутренним наполнением 48 + 4 округление для кратности 8).

Для того что бы добится вызова сборщика мусора, установил максимальный размер кучи в 3мб и
создал ~ 3350 объектов на которые никто не ссылается, после этого ВМ запускает сборщик мусора.
При запуске сборщика мусора "живые" объекты маркируются, не маркированные объекты будут удалены,
у маркированных будет изменен "счетчик возраста" и они будут перенесены из Эдема в S0 или S1.
